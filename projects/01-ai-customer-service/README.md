# AI 客服系统评测项目

> **项目定位**: 电商场景下 AI 客服多规则合规评测方案
> **验证模型**: 文心一言 ERNIE-4.5-Turbo-128K
> **项目状态**: 阶段二进行中（数据扩充与批次管理）

---

## 项目背景

### 业务场景

电商平台部署 AI 客服后，需要持续监控其回复质量，确保：
- 不越界回答（超出服务范围）
- 态度专业礼貌（无不当言辞）
- 内容真实准确（不编造信息）

### 核心挑战

| 挑战 | 传统方案 | 问题 |
|------|----------|------|
| 多规则校验 | 规则匹配（正则/关键词） | 覆盖不全、维护成本高 |
| 主观态度判定 | 人工抽查 | 效率低、标准不统一 |
| 内容真实性 | 难以自动化 | 编造信息难识别 |

### 项目目标

- ✅ 设计可复用的 Prompt 评测模板
- ✅ 开发自动化评测脚本
- ✅ 建立测试用例库
- 🚧 扩充用例到 50+ 条（进行中）
- 📅 完成 Bad Case 分析（待补充）

## 技术方案

### 方案选型

| 方案 | 优点 | 缺点 | 选择 |
|------|------|------|------|
| 规则匹配 | 速度快、成本低 | 覆盖不全、维护难 | ❌ |
| Prompt 评测 | 灵活、可扩展、支持主观判定 | 依赖模型能力 | ✅ |
| 训练专用模型 | 准确率高 | 数据需求大、成本高 | 📅 后续优化 |

**选择理由**：Prompt 评测在少量用例下即可快速验证，且可迁移性强。

### 核心设计

#### 评测维度

```
服务范围 → 是否超出客服职责范围
服务态度 → 是否礼貌、专业、无不当言辞
内容真实性 → 是否符合官方业务规则、不编造信息
```

#### Prompt 设计原则

1. **规则刚性化**：可量化、无歧义、非黑即白
2. **Few-shot 单测点**：每个示例只覆盖一个规则点
3. **格式统一**：示例格式 = 输出格式，支持脚本解析
4. **CoT 强制校验**：多规则场景分步校验，避免跳步漏检

#### 技术架构

```
测试用例库 (Markdown)
      ↓
自动化脚本 (Python + 文心一言 API)
      ↓
Prompt 模板库 (3 个已验证模板)
      ↓
评测结果 (Markdown 报告)
```

> **注**: 详细测试结果请查看各批次目录中的 `summary.md`

### CoT 对比实验

| 方法 | 用例数 | 正确率 | 推理可追溯性 | Token 成本 |
|------|--------|--------|-------------|-----------|
| No CoT | 2 | 100% | 无 | 约 609 字符 |
| CoT | 2 | 100% | 完整 | 约 996 字符 |

**关键发现**:
- 简单场景下，No CoT 与 CoT 正确率无差异
- CoT 优势：推理过程可审计、可追溯
- 建议：复杂场景（多规则冲突）下 CoT 优势更明显

```bash

# 2. 安装依赖（如果需要）
# 项目使用 Python 内置库，通常无需安装额外依赖
# 如果需要使用其他 LLM SDK，可运行：
# cd ../..  # 回到项目根目录
# pip install -r requirements.txt

# 4. 查看最新测试报告
cat results/batch-*/summary.md

# 或查看特定批次
cat results/batch-001_2026-04-05/summary.md
```
