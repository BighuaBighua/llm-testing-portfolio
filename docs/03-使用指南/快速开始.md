# 快速开始指南

> 5分钟上手 LLM 自动化评测框架

## 🚀 环境准备

### 系统要求
- **Python**: 3.8 或更高版本
- **操作系统**: Windows / macOS / Linux
- **内存**: 至少 4GB RAM
- **网络**: 稳定的互联网连接

### 安装依赖

```bash
# 克隆项目
git clone https://github.com/your-username/llm-testing-portfolio.git
cd llm-testing-portfolio

# 安装依赖
pip install -r requirements.txt

# 验证安装
python -c "import requests; print('环境准备完成')"
```

### API 密钥配置

1. **复制配置模板**
```bash
cp configs/api_config.example.yaml configs/api_config.yaml
```

2. **编辑配置文件**
编辑 `configs/api_config.yaml`，填入你的真实 API 密钥：

```yaml
# API 配置
api_config:
  openai:
    api_key: "your-openai-api-key-here"
    base_url: "https://api.openai.com/v1"
    
  anthropic:
    api_key: "your-anthropic-api-key-here"
    
  rate_limits:
    requests_per_minute: 60
    max_retries: 3
    timeout_seconds: 30
```

## 🎯 第一个评测项目

### 1. 选择评测场景

项目支持多种业务场景，首次使用建议选择默认场景：

```bash
# 查看可用场景
python scripts/generate_test_cases.py --list-scenarios

# 输出示例
可用场景:
- default (通用客服场景)
- bank (银行客服场景) 
- insurance (保险客服场景)
```

### 2. 生成测试用例

```bash
# 生成默认场景的测试用例
python scripts/generate_test_cases.py --scenario default

# 输出示例
✅ 测试用例生成完成
📁 保存位置: projects/01-ai-customer-service/cases/universal.json
📊 生成用例: 50 条
```

### 3. 执行评测

```bash
# 执行自动化评测
python scripts/run_tests.py --batch 001

# 输出示例
🚀 开始执行评测批次: batch-001
📊 测试用例总数: 50
⏰ 预计完成时间: 10-15分钟
...
✅ 评测完成: 50/50 成功
📈 通过率: 92%
📁 结果保存: projects/01-ai-customer-service/results/batch-001
```

### 4. 查看评测结果

```bash
# 查看评测摘要
cat projects/01-ai-customer-service/results/batch-001/summary.md

# 查看详细报告
cat projects/01-ai-customer-service/results/batch-001/audit_report.md
```

## 📊 理解评测结果

### 评测报告结构

每个评测批次包含以下文件：

```
results/batch-001/
├── summary.md           # 评测摘要
├── audit_report.md      # 详细审计报告
├── results.json         # 原始评测结果
├── records.json         # 执行记录
├── test_config.json     # 测试配置
└── test_execution.log   # 执行日志
```

### 关键指标解读

| 指标 | 含义 | 理想值 |
|------|------|--------|
| **通过率** | 合规测试用例比例 | > 90% |
| **平均得分** | 综合评测分数 | > 85 |
| **安全风险** | 检测到的安全问题 | 0 |
| **执行时间** | 评测耗时 | 根据用例数量 |

## 🔧 自定义配置

### 修改业务规则

编辑 `configs/business_rules.yaml` 自定义评测标准：

```yaml
# 业务规则配置
scenarios:
  default:
    name: "通用客服场景"
    service_boundaries:
      in_scope:
        - "订单查询"
        - "物流跟踪"
        - "退款申请"
      out_of_scope:
        - "产品推荐"
        - "投资建议"
    
    constraints:
      - "必须使用礼貌用语"
      - "不得泄露客户隐私"
      - "不得提供虚假信息"
```

### 调整评测维度

编辑 `configs/test_generation_config.yaml` 自定义评测重点：

```yaml
# 评测维度配置
dimensions:
  - name: "合规性测试"
    weight: 0.4
    cases_per_dimension: 10
    
  - name: "安全性测试"  
    weight: 0.3
    cases_per_dimension: 8
    
  - name: "专业性测试"
    weight: 0.3
    cases_per_dimension: 6
```

## 🚨 常见问题

### Q: API 调用失败怎么办？
**A**: 检查 API 密钥配置和网络连接，确保 API 服务可用。

### Q: 评测结果不理想如何优化？
**A**: 参考 [Bad Case 分析方法论](../04-最佳实践/Bad Case分析方法论.md) 进行问题定位。

### Q: 测试执行中断如何恢复？
**A**: 使用增量模式恢复执行，参考 [中断恢复操作指南](../04-最佳实践/中断恢复操作指南.md)。

### Q: 如何添加新的评测维度？
**A**: 在配置文件中新增维度定义，系统会自动支持。

## 📈 进阶使用

### 批量评测

```bash
# 执行多个批次评测
for i in {001..005}; do
    python scripts/run_tests.py --batch $i
    echo "批次 $i 完成"
done
```

```bash
# 监控评测性能
python scripts/run_tests.py --batch 001 --monitor

# 输出性能指标
📊 平均响应时间: 2.3s
📈 并发处理: 5 个线程
💾 内存使用: 256MB
```

```yaml

```
