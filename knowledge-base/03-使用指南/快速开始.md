# 快速开始指南

> 5分钟上手 LLM 自动化评测框架

## 🚀 环境准备

### 系统要求
- **Python**: 3.8 或更高版本
- **操作系统**: Windows / macOS / Linux
- **内存**: 至少 4GB RAM
- **网络**: 稳定的互联网连接

### 安装依赖

```bash
git clone https://github.com/BighuaBighua/llm-testing-portfolio.git
cd llm-testing-portfolio

pip install -r requirements.txt

python -c "import requests; print('环境准备完成')"
```

### API 密钥配置

1. **复制配置模板**
```bash
cp configs/api_config_example.yaml configs/api_config.yaml
```

2. **编辑配置文件**
编辑 `configs/api_config.yaml`，填入你的真实 API 密钥：

```yaml
case_generator:
  name: "百度千帆"
  model: "ernie-4.5-turbo-128k"
  base_url: "https://qianfan.baidubce.com/v2/chat/completions"
  ak: "YOUR_QIANFAN_ACCESS_KEY"
  sk: "YOUR_QIANFAN_SECRET_KEY"
  fallback:
    enable: true
    providers:
      - name: "阿里云DashScope"
        model: "qwen-turbo"
        base_url: "https://dashscope.aliyuncs.com/compatible-mode/v1"
        api_key: "YOUR_DASHSCOPE_API_KEY"

model_under_test:
  name: "百度千帆"
  model: "ernie-4.5-turbo-128k"
  base_url: "https://qianfan.baidubce.com/v2/chat/completions"
  ak: "YOUR_QIANFAN_ACCESS_KEY"
  sk: "YOUR_QIANFAN_SECRET_KEY"

evaluator:
  name: "阿里云DashScope"
  model: "qwen-turbo"
  base_url: "https://dashscope.aliyuncs.com/compatible-mode/v1"
  api_key: "YOUR_DASHSCOPE_API_KEY"
  fallback:
    enable: true
    providers:
      - name: "魔搭社区"
        model: "Qwen/Qwen3.5-35B-A3B"
        base_url: "https://api-inference.modelscope.cn/v1/"
        api_key: "YOUR_MODELSCOPE_API_KEY"
```

## 🎯 第一个评测项目

### 1. 生成测试用例

```bash
python3 scripts/generate_test_cases.py

python3 scripts/generate_test_cases.py --dimensions accuracy compliance prompt_injection

python3 scripts/generate_test_cases.py --scenario default --project 01-ai-customer-service

python3 scripts/generate_test_cases.py --append
```

支持的12个评测维度：

| 分组 | 维度 | 说明 |
|------|------|------|
| 标准维度 | accuracy, completeness, compliance, attitude, multi, boundary, conflict, induction, multi_turn | 质量评估 |
| 安全维度 | prompt_injection, sensitive_topic, bias_fairness | 安全防御 |

### 2. 执行评测

```bash
python3 scripts/run_tests.py --mode full --project 01-ai-customer-service

python3 scripts/run_tests.py --mode single

python3 scripts/run_tests.py --mode incremental --batch-id batch-003 --report append
```

### 3. 创建新项目

```bash
python3 scripts/init_project.py --name 03-medical-assistant --display "医疗助手"
```

### 4. 查看评测结果

```bash
cat projects/01-ai-customer-service/results/batch-001/audit_report.md

cat projects/01-ai-customer-service/results/batch-001/security_report.md
```

## 📊 理解评测结果

### 评测报告结构

```
results/batch-001/
├── test_config.json         # 配置基线
├── test_execution.log       # 执行日志
├── records.json             # 执行记录
├── results.json             # 评测结果
├── audit_report.md          # 审计报告
├── bug_list.md              # Bug清单
├── bypass_stats_report.md   # 绕过成功率统计
├── security_report.md       # 安全专项报告
├── evaluation_detail.csv    # 评测明细CSV
└── evaluation_summary.csv   # 统计汇总CSV
```

### 关键指标解读

| 指标 | 含义 | 理想值 |
|------|------|--------|
| **通过率** | 通过/不通过判定的通过比例 | > 90% |
| **质量门禁** | 通过率是否达到阈值 | PASS |
| **防御率** | Prompt注入攻击防御成功比例 | > 80% |
| **拦截率** | 敏感话题拦截成功比例 | > 80% |
| **公平性合规率** | 无偏见回复比例 | > 90% |
| **安全评级** | 综合安全风险等级 | 低风险 |

## 🔧 自定义配置

### 修改业务规则

编辑 `configs/business_rules.yaml`：

```yaml
active_scenario: "default"

scenarios:
  default:
    name: "通用客服"
    description: "回答用户关于服务、流程、操作等方面的问题"
    service_boundaries:
      in_scope:
        - "订单查询"
        - "物流跟踪"
      out_of_scope:
        - "投资建议"
    constraints:
      - "必须使用礼貌用语"
      - "不得泄露客户隐私"
```

### 调整执行参数

编辑 `configs/execution.yaml`：

```yaml
concurrency:
  default_mode: "concurrent"

parameters:
  timing:
    case_timeout: 300
  inference:
    under_test:
      temperature: 0.7
    evaluator:
      temperature: 0.3

quality_gate:
  overall_threshold: 0.9
```

## 🚨 常见问题

### Q: API 调用失败怎么办？
**A**: 系统支持 Fallback 机制，在 `api_config.yaml` 中配置备用 Provider 即可自动切换。

### Q: 评测结果不理想如何优化？
**A**: 参考 [Bad Case 分析方法论](../04-最佳实践/Bad%20Case分析方法论.md) 进行问题定位。

### Q: 测试执行中断如何恢复？
**A**: 参考 [中断恢复操作指南](../04-最佳实践/中断恢复操作指南.md)。

### Q: 如何添加新的评测维度？
**A**: 在 `configs/test_generation.yaml` 中新增维度定义，并创建对应的生成模板。
