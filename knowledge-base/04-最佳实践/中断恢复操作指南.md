# 中断恢复操作指南

> 基于 TestRunRecorder 中断检测的断点续传操作指南

## 🎯 中断场景概述

### 常见中断原因

| 中断原因 | 特征 | 恢复策略 |
|----------|------|----------|
| 网络超时 | API 调用超时 | 重试失败用例 |
| API 限流 | 429 Too Many Requests | 等待冷却后继续 |
| 进程崩溃 | Python 异常退出 | 从断点继续 |
| 手动中断 | Ctrl+C 终止 | 从断点继续 |
| 系统资源不足 | 内存/磁盘不足 | 释放资源后继续 |

### 中断检测机制

系统通过 `TestRunRecorder.detect_interruption()` 自动检测中断：

```python
def detect_interruption(self) -> Dict[str, Any]:
    """检测批次是否因中断而未完成"""
    if self.config["status"] == "completed":
        return {"detected": False, "reason": "Batch already completed"}

    last_completed = self.get_last_completed_case()
    total = self.config["test_configuration"]["total_cases"]

    if last_completed is None:
        return {"detected": True, "completed": 0, "total": total, "last_completed": None}

    completed_count = sum(1 for line in open(self.log_file) if "completed" in line)
    return {
        "detected": completed_count < total,
        "completed": completed_count,
        "total": total,
        "last_completed": last_completed,
        "last_timestamp": last_timestamp
    }
```

## 🔧 恢复操作流程

### 1. 检测中断状态

```python
from tools.execution import TestRunRecorder

recorder = TestRunRecorder("projects/01-ai-customer-service/results/batch-016_2026-04-13")
interruption = recorder.detect_interruption()

if interruption["detected"]:
    print(f"检测到中断！已完成 {interruption['completed']}/{interruption['total']}")
    print(f"最后完成的用例: {interruption['last_completed']}")
else:
    print("批次已完成，无需恢复")
```

### 2. 版本兼容性检查

恢复前需检查用例版本是否与批次配置一致：

```python
compatibility = recorder.check_version_compatibility(current_case_version="2.1")

if not compatibility["compatible"]:
    print(f"版本不兼容！批次版本: {compatibility['batch_version']}, 当前版本: {compatibility['current_version']}")
    print("建议重新生成用例或使用对应版本的用例")
```

### 3. 手动恢复步骤

```bash
# 1. 查看批次状态
cat projects/01-ai-customer-service/results/batch-016_2026-04-13/test_config.json | grep status

# 2. 查看执行日志，确认最后完成的用例
tail -20 projects/01-ai-customer-service/results/batch-016_2026-04-13/test_execution.log

# 3. 查看已完成的结果数量
python3 -c "
import json
results = json.load(open('projects/01-ai-customer-service/results/batch-016_2026-04-13/results.json'))
print(f'已完成: {len(results)} 条')
"

# 4. 重新运行评测（增量模式，跳过已完成的用例）
python3 scripts/run_tests.py --mode incremental --batch-id batch-016 --report append
```

### 4. 配置完整性验证

恢复前验证配置基线完整性：

```python
recorder = TestRunRecorder(batch_dir)
integrity = recorder.validate_config_integrity()

if not integrity["passed"]:
    print(f"配置基线不完整，缺失字段: {integrity['missing_fields']}")
```

## 📋 恢复检查清单

### 恢复前检查

- [ ] 确认中断原因已解决（网络/API/资源）
- [ ] 检查批次状态（`test_config.json` → `status`）
- [ ] 确认已完成用例数量（`results.json` 长度）
- [ ] 检查用例版本兼容性（`check_version_compatibility`）
- [ ] 验证配置基线完整性（`validate_config_integrity`）

### 恢复后验证

- [ ] 确认所有用例已执行完成
- [ ] 验证执行记录与评测结果数量一致（`validate_consistency`）
- [ ] 验证用例覆盖率（`validate_coverage`）
- [ ] 查看审计报告确认完整性

## 🚨 常见问题

### Q: 中断后如何确认哪些用例已完成？
**A**: 查看 `results.json` 中的记录，每条记录包含 `id` 字段，可对比 `universal.json` 中的用例列表确认缺失项。

### Q: 版本不兼容时如何处理？
**A**: 如果 `check_version_compatibility()` 返回不兼容，建议重新生成用例并重新执行评测，避免用例版本与评测标准不一致。

### Q: 如何避免中断？
**A**: 
- 配置合理的超时参数（`execution.yaml` → `parameters.timing`）
- 启用 API Fallback 机制
- 使用并发模式而非性能模式（减少资源压力）
- 确保网络稳定
